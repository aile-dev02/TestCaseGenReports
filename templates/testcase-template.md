# テンプレートの使い方

## コピーして使う手順

1. このファイルを `master/testcases/` にコピーする
2. ファイル名を `{カテゴリ接頭辞}-{連番3桁}.md` に変更する（例: `AUTH-002.md`）
3. FrontMatter の各フィールドを埋める
4. `# 手順` と `# 期待結果` を記述する
5. この説明セクション（`---` 以下）を削除する
6. `npm run validate` で検証が通ることを確認する

---

## フィールド説明

| フィールド | 必須 | 説明 | 設定例 |
|:----------|:----:|:-----|:------|
| `id` | ✅ | 一意の識別子。`{接頭辞}-{3桁番号}` 形式 | `AUTH-001` |
| `タイトル` | ✅ | テストケースの内容を端的に表すタイトル | `正常ログイン` |
| `要件ID` | 推奨 | 紐づく要件の ID（`REQ-CATEGORY-NNN` 形式） | `REQ-LOGIN-001` |
| `優先度` | ✅ | 実行優先度。下の表を参照 | `high` |
| `カテゴリ` | ✅ | 機能カテゴリ・モジュール名（英数字小文字推奨） | `auth` |
| `タイプ` | ✅ | テスト設計タイプ。下の表を参照 | `positive` |
| `前提条件` | 任意 | テスト実行前に満たすべき状態の一覧 | ユーザーが存在する |
| `タグ` | 任意 | フィルタリング用の自由タグ（複数可） | `smoke`, `regression` |

---

## 優先度の選び方

| 値 | 意味 | 目安 |
|:--|:-----|:-----|
| `high` | 高優先度 | リリースブロッカーになりうる機能。スモークテストに含める |
| `medium` | 中優先度 | 主要機能だが障害時の影響が限定的 |
| `low` | 低優先度 | 補助的な機能・エッジケース |

---

## タイプの選び方

| 値 | 意味 | 例 |
|:--|:-----|:---|
| `positive` | 正常系 | 正しい入力で期待通りの結果が得られる |
| `negative` | 異常系 | 不正な入力・操作に対して適切にエラーになる |
| `boundary` | 境界値 | 最大値・最小値・境界前後の値を検証する |
| `security` | セキュリティ | 認証・認可・インジェクション等のセキュリティ検証 |
| `performance` | 性能 | レスポンスタイム・スループット等の性能検証 |

---

## ファイル名・ID の命名規則

```
ファイル名: {カテゴリ接頭辞}-{3桁連番}.md
ID:        {カテゴリ接頭辞}-{3桁連番}

例:
  auth/    → AUTH-001.md   id: AUTH-001
  user/    → USER-003.md   id: USER-003
  payment/ → PAYMENT-001.md  id: PAYMENT-001
```

> **注意:** ファイル名と `id` フィールドは必ず一致させてください。

## 記述例（完成形）

```markdown

id: AUTH-001
タイトル: 正常ログイン
要件ID: REQ-LOGIN-001
優先度: high

前提条件:
  - ユーザーアカウントが存在する
  - アカウントが有効（停止・削除されていない）

  - smoke

# 手順

1. ブラウザでログイン画面を開く
2. 有効なユーザーIDを入力する
3. 正しいパスワードを入力する
4. ログインボタンを押下する

# 期待結果

- マイページ（/dashboard）へリダイレクトされる
- セッションCookieが生成される
- ヘッダーにユーザー名が表示される
```

## よくある間違いと対策

| 間違い | 原因 | 対策 |
|:------|:-----|:-----|
| バリデーションエラー `DUPLICATE_ID` | 既存 ID と重複している | `master/testcases/` の最大番号を確認して採番する |
| バリデーションエラー `MISSING_手順` | `# 手順` の見出しが違う・リスト項目がない | 見出し名を `# 手順` に統一し、`1.` から始まるリストを書く |
| バリデーションエラー `MISSING_期待結果` | `# 期待結果` の見出しが違う・リスト項目がない | 見出し名を `# 期待結果` に統一し、`-` から始まるリストを書く |
| スキーマエラー `優先度` | `High` など大文字で入力している | `high` / `medium` / `low` のいずれかを小文字で指定する |
